시스템 기본 개요

본 시스템은 요양원 운영 실무 흐름을 기준으로 설계된 프리미엄 관리자용 테스트 버전입니다. 표지 화면, 개요 페이지, 입력 페이지, 출력 제목별 페이지, 상세 사용문서 페이지를 단계적으로 분리하여 첫 접속 시 답답함을 줄이고 화면 집중도를 높였습니다.

시스템 목적

입소자 상태기록을 구조화하고, 보고서 출력, 보호자 안내, 운영관리, 이력 추적을 하나의 흐름으로 연결하는 데 목적이 있습니다.

주요 특징

입력과 출력 분리, 모바일 대응, 고급형 표지 화면, 관리자 중심 페이지 구조, 실무형 다운로드 기능을 포함합니다.

운영관리 기능

입소자 대시보드, 고위험 인원 관리, 보호자 연락처 관리, 전송이력 기록, 입소자별 개별 히스토리를 지원합니다.

분석 기능

식사량과 야간 호출 추세, 위험등급 분포, 최근 상태 해석 요약을 그래프로 확인할 수 있도록 설계했습니다.

입력 기본 원칙

각 항목은 관찰된 사실 중심으로 입력합니다. 추정 진단이나 해석적 표현보다는 관찰 내용, 빈도, 정도, 시간 정보를 중심으로 기재하는 것이 좋습니다.

식사 및 배설 입력 가이드

식사량은 % 또는 실제 섭취 정도로 일관되게 입력합니다. 배변 상태는 있음, 없음, 횟수, 평소 대비 변화 여부를 함께 기록하면 추적에 유리합니다.

이동 및 수면 입력 가이드

보행 상태는 안정, 약간 불안정, 불안정처럼 구간화해 입력하면 비교가 쉽습니다. 수면 상태는 보통, 약간 저하, 수면 저하처럼 동일 표현체계를 유지하는 것이 바람직합니다.

활력징후 및 증상 입력 가이드

체온, 맥박, 혈압은 숫자 형식을 가능한 한 일정하게 유지합니다. 기침, 통증, 호흡 상태는 빈도와 양상을 중심으로 기록하면 보고서 정확도가 높아집니다.

저장 안내 한눈에 보기

이 시스템은 같은 PC와 같은 브라우저에서 자동으로 기록을 보관합니다. 파일을 다시 열어도 직전 입력 내용, 입소자 현황, 전송이력, 히스토리가 남아 있도록 구성했습니다.

      본 페이지는 시스템 구조, 사용매뉴얼, 주의사항, 권리관계 고지사항을 포함한 최종 안내 문서입니다. 요청하신 바에 따라 글자 크기는 다른 페이지보다 작게 적용했습니다.

      1. 시스템 구조

      본 시스템은 표지 화면, 시스템 개요 안내 화면, 기본 입력화면, 출력 제목별 운영 페이지, 최종 사용설명 페이지의 순서로 구성됩니다. 첫 화면에서 과도한 정보가 한꺼번에 노출되지 않도록 설계하여 사용자의 피로감을 줄이고, 단계별 흐름에 따라 기능을 이해하도록 구성했습니다.

      2. 사용 매뉴얼

      • 표지 화면에서 시스템 시작 또는 바로 입력하기 버튼을 눌러 진입합니다.
      • 개요 페이지에서 시스템 목적과 특징을 확인합니다.
      • 입력 페이지에서 입소자 상태 기록을 입력합니다.
      • 기본 출력화면 생성 버튼을 누르면 상태 보고서 페이지로 이동하며 출력 내용이 갱신됩니다.
      • 이후 대시보드, 프롬프트, 외부 전달, 전송이력, 히스토리, 통계분석 페이지를 순차적으로 활용합니다.
      • 필요 시 화면 파일 저장, 전체 기록 백업 저장, 각종 엑셀용 저장 기능을 이용합니다.

      저장 방식 쉬운 설명

      자동 보관은 같은 PC와 같은 브라우저 안에 기록이 남는 기능입니다. 엑셀용 저장은 표를 엑셀에서 열기 위한 저장입니다. 전체 기록 백업 저장은 시스템의 주요 기록을 한 번에 묶어 보관하는 기능입니다. 화면 파일 저장은 현재 앱 화면 자체를 사본으로 보관하는 기능입니다.

      3. 주의사항

      • 본 시스템은 기록관리 및 운영보조용 테스트 화면입니다.
      • 자동 생성 문구는 사실 입력값을 정리한 참고자료이며, 의학적 진단이나 법적 판단 자료를 대체하지 않습니다.
      • 개인정보 입력 시에는 내부 보안기준과 비식별화 원칙을 우선 적용해야 합니다.
      • 전송 기능은 로컬 HTML 환경 기준으로 초안 생성과 복사 중심이며, 자동 발송 시스템은 별도 서버 연동이 필요합니다.
